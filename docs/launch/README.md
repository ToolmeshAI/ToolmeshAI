# Launch Kit

## English

This folder contains the first outbound distribution kit for the `ToolmeshAI` portfolio.

What is here:

- `launch-queue.md`: operator runbook with platform status, posting order, direct submission links, and repo-by-repo queue
- `x.md`: short launch posts for X
- `reddit.md`: post ideas for Reddit-style communities
- `hacker-news.md`: titles and launch angles for Hacker News
- `zh-communities.md`: Chinese launch copy for developer communities
- `generate_assets.py`: local asset generator for social preview PNGs and demo GIFs

Recommended posting order:

1. `mcp-saas-foundry`
2. `mcp-smoke-test`
3. `safe-mcp-config`
4. `docs-to-context`
5. `browser-agent-starter`
6. `ToolmeshAI`
7. `awesome-mcp-workflows`
8. `browser-agent-recipes`

Asset usage:

- Upload each repo's `docs/assets/social-preview.png` as the GitHub social preview image manually in repository settings.
- Use each repo's `docs/assets/demo.gif` inside README sections and in social posts where animation helps.
- Prefer one hero asset per post, not multiple competing screenshots.

## 中文

这个目录放的是 `ToolmeshAI` 第一轮对外分发包。

内容包括：

- `launch-queue.md`：执行队列文档，包含平台状态、发帖顺序、直达发布链接和按仓库拆分的队列
- `x.md`：适合发到 X 的短帖文案
- `reddit.md`：适合 Reddit 类社区的发帖文案
- `hacker-news.md`：适合 Hacker News 的标题和角度
- `zh-communities.md`：适合中文开发者社区的文案
- `generate_assets.py`：本地生成社交预览图和 demo GIF 的脚本

建议发布顺序：

1. `mcp-saas-foundry`
2. `mcp-smoke-test`
3. `safe-mcp-config`
4. `docs-to-context`
5. `browser-agent-starter`
6. `ToolmeshAI`
7. `awesome-mcp-workflows`
8. `browser-agent-recipes`

素材使用建议：

- 各仓库的 `docs/assets/social-preview.png` 建议手动上传到 GitHub 仓库设置里的 social preview。
- 各仓库的 `docs/assets/demo.gif` 适合放进 README，也适合发社交平台时做第一屏素材。
- 一条帖子尽量只打一个主视觉，不要堆多个截图互相抢注意力。
