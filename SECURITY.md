# Security Policy

This policy is the default for public repositories under the ToolmeshAI account unless a repository ships its own `SECURITY.md`.

## Scope

- Public code, docs, examples, and automation assets published under `github.com/ToolmeshAI`
- The latest default branch and the newest tagged release of an affected repository

Older snapshots may not receive backported fixes unless the risk is severe and the maintainer has capacity.

## Reporting

- Do not open a public issue with exploit details, secrets, or a working proof of concept.
- If the affected repository supports private vulnerability reporting, use that first.
- If private reporting is not available, open a minimal public issue that only asks for a private contact path. Do not include sensitive details.

Include the repository name, affected version or commit, impact, reproduction steps, and any known mitigation.

## Response

Security triage is handled on a best-effort basis.

- Initial acknowledgement target: within 5 business days
- Status updates: shared after triage when a fix path is clear
- Disclosure: coordinated disclosure is preferred once affected users have a reasonable upgrade path
